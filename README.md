# AMD MI355X GPU算子优化

## 开源说明

项目初期使用Git和GitHub进行代码版本管理。随着部分算子成绩进入排行榜前列，考虑到公开仓库可能被他人检索发现，在比赛后期主动移除了该项目。比赛现已结束，秉持开源精神与知识共享的理念，重新发布优化代码与心得，希望能为GPU算子优化领域的开发者提供参考，促进技术交流与共同进步。

## 成绩排名

本项目为AMD GPU MODE Hackathon参赛作品，在MI355X GPU上优化设计了三个关键推理算子。以下为取得排名的算子成绩：

### MM算子（MXFP4矩阵乘法）
- 排名：第6名
- 延迟：7.934μs (M=4, N=2880, K=512)

### MoE算子（MXFP4专家混合）
- 排名：第5名
- 延迟：90.8μs (bs=16, E=257)

## 优化内容

### 1. MM算子
利用AMD硬件量化指令`v_cvt_scalef32_pk_fp4_bf16`实现BF16→MXFP4融合量化，配合Split-K并行策略和预洗牌权重布局。

详见：[mm算子优化思路与心得.md](mm算子优化思路与心得.md)

### 2. MoE算子
针对不同batch和专家数量选择最优block_m参数，采用两阶段流水线和持久化Buffer池减少内存开销。

详见：[moe算子优化思路与心得.md](moe算子优化思路与心得.md)

### 3. MLA算子
实现FP8/BF16混合精度解码路径，根据batch大小动态选择KV分片数和页大小配置。该算子参与了比赛优化实践。

详见：[mla算子优化思路与心得.md](mla算子优化思路与心得.md)
